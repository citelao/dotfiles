Today, callers can’t safely perform division through the math utility library because it does not provide a built-in divide helper. This change fixes that by adding a `divide` function to the library.

**What Changed**
- Added a new `divide` function to the math utility library.
- Exposed the function through the library’s public API so callers can use it alongside the existing math helpers.
- Defined expected division behavior for valid numeric inputs.

**Test Plan**
- [ ] Verify `divide(10, 2)` returns `5`.
- [ ] Verify `divide(-9, 3)` returns `-3`.
- [ ] Verify `divide(5, 2)` returns `2.5`.
- [ ] Verify division-by-zero behavior matches the library’s intended contract.
- [ ] Run existing unit test suite for the math utility library.
