Today, the math utility library lacks a divide function, forcing callers to inline division logic. This change adds a `divide` function that handles division cleanly and raises on divide-by-zero.

**What Changed**

- Added `divide(a, b)` function to the math utility library
- Raises an exception when `b` is zero rather than returning `Infinity` or `NaN`

**Test Plan**

- [ ] `divide(10, 2)` returns `5`
- [ ] `divide(7, 3)` returns the correct decimal result
- [ ] `divide(x, 0)` raises an exception
- [ ] Existing math utility tests still pass
